# Reglas y Estructura del Proyecto: AM1 Lacabanne

Este archivo define las directrices de diseño, la estructura del proyecto y las reglas de codificación y contenido para desarrolladores y asistentes de IA en el repositorio **AM1 Lacabanne**.

---

## 📝 Descripción del Proyecto

**AM1 Lacabanne** es la plataforma web de documentación y aula virtual para la materia **Artes Multimediales 1 (Cátedra Lacabanne)** de la **Universidad Nacional de las Artes (UNA)**, Argentina. Sirve como guía de taller para el aprendizaje de **p5.js** (programación creativa), control de actividades y consignas de los trabajos prácticos.

---

## 🛠️ Stack Tecnológico

*   **Framework:** Astro `^4.3.5`
*   **Tema/Documentación:** Starlight `^0.21.2`
*   **Lenguaje:** TypeScript `^5.4.3` (configuración estricta en [tsconfig.json](file:///D:/code/am1-lacabanne/tsconfig.json))
*   **Integraciones:** `@astro-community/astro-embed-youtube` para la inserción optimizada de clases y recursos en video.

## ⚠️ Reglas Críticas de Contenido y Nomenclatura

### 1. Nomenclatura de Archivos (Kebab-Case)
Todos los archivos y directorios creados dentro de la estructura del proyecto **deben obligatoriamente** utilizar formato **kebab-case**:
*   Sin espacios.
*   Sin mayúsculas.
*   Sin acentos ni tildes (ej: `guia-montaje-tp1.mdx`, **no** `guía-montaje-tp1.mdx`).
*   Sin la letra `ñ` ni caracteres especiales.

### 2. Recursos y Multimedia Locales
Las imágenes utilizadas por las clases deben guardarse localmente en el directorio de la clase respectiva y cargarse utilizando la sintaxis MDX de importación de Astro:
```mdx
import img1 from './mi-imagen.png';
import { Image } from 'astro:assets';

<Image src={img1} alt="Descripción de la imagen" width="700"/>
```

### 3. Embeber Videos de YouTube
Para insertar videos explicativos o de clases grabadas, se debe utilizar el componente `<YouTube />`:
```mdx
import { YouTube } from '@astro-community/astro-embed-youtube';

<YouTube id="https://youtu.be/ID_DEL_VIDEO" params="start=0&end=500"/>
```

## 🎓 Convenciones para Entregas de Alumnos (`la-piscine`)

Los alumnos entregan sus trabajos prácticos realizando un Pull Request al repositorio de la galería `<la-piscine>`. Cada entrega debe respetar el formato establecido en la [guia-montaje-tp1.mdx](file:///D:/code/am1-lacabanne/src/content/docs/clases/misc/guia-montaje-tp1.mdx):
1.  **Carpeta única:** `<año>-tp<n>-<apellido>-<nombre>` (kebab-case estricto).
2.  **Archivos requeridos:**
    *   `thumbnail.webp` (Miniatura obligatoria de `360x360` píxeles).
    *   `sketch.js` (Código p5.js, donde el canvas debe enlazarse mediante `canvas.parent('div-sketch')`).
    *   `statement.mdx` (Declaración bilingüe estructurada usando las etiquetas `<Language code="es">` y `<Language code="en">`).

## 🚀 Comandos de Ejecución y Despliegue

| Comando | Descripción |
| :--- | :--- |
| `npm install` | Instala los módulos y dependencias de Node. |
| `npm run dev` | Inicia el servidor de desarrollo local en `http://localhost:4321`. |
| `npm run build` | Compila y genera el sitio estático para producción en `/dist/` validando tipos. |
| `npm run preview` | Permite visualizar localmente el sitio compilado en `/dist/`. |
